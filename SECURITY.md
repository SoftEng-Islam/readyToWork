# Security Policy

## Supported Versions

Security fixes should target the active default branch first. If you publish release branches later, document support windows here.

## Reporting A Vulnerability

Do not open a public issue for security problems.

Use GitHub's private vulnerability reporting flow if it is enabled for the repository. If private reporting is not available yet, contact the repository maintainer directly before disclosing details publicly.

When you report an issue, include:

- A short summary of the problem
- Steps to reproduce it
- The affected area or endpoint
- Any proof-of-concept or logs that help confirm impact

We recommend rotating any exposed credentials immediately if the report involves secrets.
